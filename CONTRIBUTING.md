# 贡献指南

感谢你考虑为本项目做出贡献！你的参与能让这个项目变得更好。请花一点时间阅读以下指南，以确保贡献过程顺利进行。

## 如何贡献

1. **Fork 仓库**
   - 点击仓库右上角的 "Fork" 按钮，创建你自己的副本。

2. **克隆你的 Fork**
   - 将你 Fork 的仓库克隆到本地：
     ```bash
     git clone https://github.com/your-username/zenscreen.git
     ```

3. **创建新分支**
   - 为你的功能或修复创建一个分支：
     ```bash
     git checkout -b feature/your-feature-name
     ```

4. **进行修改**
   - 进行你的修改。

5. **测试你的修改**
   - 彻底测试你的修改，确保它们按预期工作，并且不会破坏现有功能。

6. **提交你的修改**
   - 使用清晰简洁的提交信息提交你的修改：
     ```bash
     git add .
     git commit -m "简要描述你的修改"
     ```

7. **推送你的修改**
   - 将你的分支推送到你 Fork 的仓库：
     ```bash
     git push origin feature/your-feature-name
     ```

8. **提交 Pull Request**
   - 前往原始仓库，从你的分支提交一个 Pull Request。清晰描述你的修改以及它们解决的问题。

## 报告问题

如果你遇到 Bug 或有功能建议，请在仓库的 [Issues](https://github.com/hwdemtv/zenscreen/issues) 部分提交 Issue。请提供尽可能详细的信息，帮助我们有效解决问题。

## 代码风格指南

- 编写清晰、简洁、描述性的提交信息。
- 在必要时添加注释来解释复杂的代码。

## 许可证

通过向本项目贡献代码，你同意你的贡献将按照 [MIT 许可证](./LICENSE) 授权。

感谢你的贡献！
